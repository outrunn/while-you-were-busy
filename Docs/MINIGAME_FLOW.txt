================================================================================
                         MINIGAME FLOW DIAGRAM
================================================================================

┌─────────────────────────────────────────────────────────────────────────────┐
│ TICKET LIFECYCLE                                                            │
└─────────────────────────────────────────────────────────────────────────────┘

1. PRINTER SPAWNS TICKET
   ↓
   Printer.PrintTicket()
   ↓
   Creates Ticket GameObject from prefab
   ↓
   Calls Ticket.Initialize(TaskData)
   ↓
   TaskData contains:
     - title
     - description
     - baseReward
     - difficultyLevel
     - minigameType (enum: Typing, MultipleChoice, Math, PhotoReveal)
     - typingTask (only for Typing type)

2. TICKET SHOWS ON BOARD
   ↓
   Displays title/description
   ↓
   Shows "START TASK" button (if minigameType != None)

3. PLAYER CLICKS "START TASK"
   ↓
   Ticket.OnStartTaskButtonClicked()
   ↓
   Switch on taskData.minigameType:
     - TYPING → TypingMinigameUI.Instance.StartMinigame()
     - MULTIPLE_CHOICE → MultipleChoiceMinigameUI.Instance.StartMinigame()
     - MATH → MathMinigameUI.Instance.StartMinigame()
     - PHOTO_REVEAL → PhotoRevealMinigameUI.Instance.StartMinigame()

4. MINIGAME RUNS
   ↓
   Each minigame:
   - Shows window
   - Listens for input
   - Validates answer
   - On correct answer:
     └─ "Correct!" message
     └─ Wait 2.5 seconds (completionDelay)
     └─ Call onMinigameCompleted callback
     └─ Hide window

   - On wrong answer:
     └─ "Incorrect!" message
     └─ Red flash effect
     └─ Clear input/reset state
     └─ Allow retry (player can try again immediately)

5. CALLBACK FIRES: Ticket.OnMinigameCompleted()
   ↓
   Calls Ticket.StampTicket()
   ↓
   Sets isStamped = true
   ↓
   Shows stamp animation/icon
   ↓
   Hides "START TASK" button
   ↓
   Logs "Task completed! Drag to processing machine."

6. PLAYER DRAGS STAMPED TICKET TO PROCESSING MACHINE
   ↓
   ProcessingMachine.OnDrop(Ticket)
   ↓
   Processes the ticket:
     - Adds reward to GameManager.outputs
     - Adds to dailyPoints (counts toward quota)
     - Degrades world health
     - Destroys ticket
     - Tells Printer to decrement activeTicketCount

================================================================================
                         TASK GENERATION LOGIC
================================================================================

When Printer.PrintTicket() is called:
  ↓
  Gets current day from GameManager.GetCurrentDay()
  ↓
  Gets difficulty from GameManager.GetDifficultyForCurrentProgress()
  ↓
  Calls TaskDatabase.GetRandomTaskForDay(day, difficulty)
  ↓
  TaskDatabase filters available tasks by day:

    DAY 1:
      ✓ minigameType == Typing
      ✗ minigameType == MultipleChoice (not allowed)
      ✗ minigameType == Math (not allowed)
      ✗ minigameType == PhotoReveal (not allowed)

    DAY 2:
      ✓ minigameType == Typing
      ✓ minigameType == MultipleChoice
      ✗ minigameType == Math
      ✗ minigameType == PhotoReveal

    DAY 3:
      ✗ minigameType == Typing
      ✗ minigameType == MultipleChoice
      ✓ minigameType == Math
      ✗ minigameType == PhotoReveal

    DAY 4:
      ✗ minigameType == Typing
      ✗ minigameType == MultipleChoice
      ✗ minigameType == Math
      ✓ minigameType == PhotoReveal

    DAY 5:
      ✓ minigameType == Typing (fallback, rarely used due to impossible quota)

  ↓
  Returns random task from filtered list
  ↓
  Ticket.Initialize(TaskData) stores the task
  ↓
  Ticket displays with correct minigame type
